# Flash Trade Challenge - Submission & Evaluation Guide

## 1. Submission Instructions (for Participants)

### A. Screen Recording (Required)

- **1 continuous screen recording** (no cuts)
- Recommended length: Less than **5 minutes**

#### Required Flow:
1. Open / Install the app
2. Buy a token successfully
    - User must have no debt
    - User must own the token
3. Sell the token (ideally after 1-2 minutes)

**Critical Requirements:**
- Clearly visible time from **app open â†’ successful buy**
- **Focus on speed and functionality**
- UI quality is NOT evaluated

### B. App Access & Testing Information

**Technical Testing Details:**
- **Platform:** iOS / Android
- **Installation Method:**
    - TestFlight
    - APK
    - Private store
    - GitHub release
- **Setup Requirements:**
    - Network configuration
    - Environment variables
    - API keys

**Wallet & Funding Specifications:**
- Wallet Type:
    - In-app wallet
    - External wallet
- Funding Method:
    - Users must self-fund wallet
    - **No borrowed or pre-funded accounts**

### C. Technical Optimization Notes

Answer these 3 critical questions (5-7 lines total):

1. **Speed Optimization Strategies:**
    - Specific techniques to accelerate first token purchase
    - Performance optimization approaches

2. **Best-Case Performance:**
    - Exact time from app open to successful buy
    - Minimum achievable transaction time

3. **Architectural Trade-offs:**
    - Key assumptions in design
    - Performance vs. complexity trade-offs

## 2. MVC Judging & Evaluation Process

### Phase 1: Kyber Internal MVC Verification

**Offline Review Checklist:**
- App functionality validation
- Buy/sell flow verification
- **Kyber Aggregator integration**

**Evaluation Outcome:**
- MVC Pass/Fail Status
- Recorded purchase time

> **Advancement Rule:** Only MVC-passing teams proceed to Phase 2

### Phase 2: Live Demo Evaluation

**Demo Call Structure:**
1. Live Demonstration (5-10 min)
    - Real-time app launch
    - Live token purchase
    - Independent time measurement

2. Technical Q&A (5-10 min)
    - Performance optimization insights
    - Identified bottlenecks
    - Production improvement strategies

3. Future Collaboration Discussion (5-10 min)

## 3. Reward & Selection Criteria

### Base Reward: $1,000 (MVC Qualification)

**Mandatory Requirements:**
- Fully functional application
- Seamless buy/sell mechanisms
- **Kyber Aggregator integration**

### Advanced Reward: $5,000

**Ranking Evaluation Factors:**
1. **Speed**
    - App open to purchase completion time
2. **Reliability**
    - Consistent transaction flow
3. **Technical Integrity**
    - No performance manipulation

### Important Evaluation Notes
- UI and branding NOT evaluated
- Network issue retries permitted
- Focus on core functional performance
